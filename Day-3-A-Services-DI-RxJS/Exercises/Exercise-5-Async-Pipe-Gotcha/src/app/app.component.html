The async Pipe Null Gotcha

Logged-in User

Welcome, {{ user$ | async }}!

What's happening (Part A)

  • The service's BehaviorSubject starts with null
  • The async pipe subscribes and immediately receives that null
  • The template renders null — literally the word "null"
  • After 1.5 seconds the real value arrives

The fix (Part B)

  • Replace async pipe with toSignal(obs$, { initialValue: 'Loading...' })
  • initialValue sets what the signal returns before the Observable emits
  • The signal never holds null unless you explicitly pass it as the initial value
  • No @if null guard needed — the template gets simpler
